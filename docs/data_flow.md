# Data Flow


## Terminology

- `ctc`: center-to-center distance between minibeams.
- `PVDR`: peak-to-valley dose ratio.
- `FWHM`: full width at half maximum.
- `PDD`: percentage depth dose.
- `PBP`: pristine Bragg peak.
- `SOBP`: spread-out Bragg peak.
- `bw`: beam width. In the legacy scripts the variable named `FWHM` is usually entered as tenths of a millimeter, for example `5` means `0.5 mm`.

## Inspected Legacy Files

The data-flow reconstruction used the active scripts and utilities in the research folder, including `utilities/getprofile.py`, `utilities/getprofile_updated.py`, `create_multibeams.py`, `fit_FWHM.py`, `fit_singlebeam_FWHM.py`, `inspect_PVDRvsctc_toreview.py`, `inspect_1Dhomogeneity.py`, `inspect_1Dhomog_and_std.py`, `inspect_2Dlateralhomogeneity.py`, `inspect_xdeviation.py`, `inspect_PDD.py`, `mergestatistics.py`, `plot_FWHMvsE.py`, `plot_ctchomgeneity.py`, and the slider scripts for dependency tracing.

## 1. Raw/Source Input Formats

The active proton pMBRT workflow is centered on 3D dose grids:

| Format | How it is used | Notes |
|---|---|---|
| `.npy` | Main input for the active proton scripts. Loaded with `numpy.load`. | The array is treated as a 3D dose grid. |
| `.header` | Optional sidecar for `.npy` dose grids. | Read as CSV with columns `res_x`, `res_y`, and `res_z`; legacy code multiplies these values by 10 to obtain millimeters. |
| `.mhd` | Supported by the base loader through SimpleITK. | Not included in the public repository. |
| TOPAS/statistics text | Used by `mergestatistics.py` to sum simulated primary counts from multiple statistics files. | Not used by the first public plotting scripts. |
| ROOT phase-space files | `utilities/readROOT.py` can read a `PhaseSpace` tree with `uproot`. | Not used by the first public plotting scripts. |

## 2. Intermediate Data Generated by Preprocessing

The active proton workflow uses derived arrays and text files before plotting:

| Step | Script/function | Input | Output |
|---|---|---|---|
| Create multi-beam dose grid | `create_multibeams.py` | `protonsinglebeam-Dose.npy` | `ctc{label}/protonmultibeam-Dose.npy` |
| Load profiles | `utilities/getprofile.py` and `utilities/getprofile_updated.py` | `.npy` or `.mhd` dose grid | In-memory `z_peak`, `z_valley`, masked `zxprofile`, masked `zyprofile` |
| Select PBP or SOBP source | `utilities/getprofile_updated.py` | `monobeam=True` or `False` | PBP uses `symbeam_PBP`; SOBP uses `symbeam_SOBP` |
| Fit single-beam FWHM | `fit_singlebeam_FWHM.py` | single-beam lateral profiles | `FWHM_singlebeam_{energy}MeV.txt` |
| Fit multi-beam FWHM | `fit_FWHM.py` | multi-beam lateral profiles | `FWHM_1DGRID_ctc{ctc}_{energy}MeV.txt` or `FWHM_2DGRID_ctc{ctc}_{energy}MeV.txt` |
| Compute PVDR | `inspect_PVDRvsctc_toreview.py`, `Slider_PVDR.py` | peak and valley depth profiles | PVDR arrays computed as `z_peak / z_valley` |
| Compute homogeneity | `inspect_1Dhomogeneity.py`, `inspect_1Dhomog_and_std.py`, `inspect_2Dlateralhomogeneity.py`, `inspect_xdeviation.py` | masked lateral profiles | homogeneity and standard-deviation `.npy` outputs |

The multi-beam creator and profile loaders use center-to-center distance labels as tenths of a millimeter, for example `ctc12` for `1.2 mm`.

## 3. Processed Numerical Data Used for Public Plotting

The first public repository exposes only manuscript Figures 1, 2, and 5.

| Figure | Public processed inputs used |
|---|---|
| Figure 1 | PBP `zpeak_1Darray_ctc<ctc>_<energy>MeV.txt` files. Each profile is normalized by its own maximum during plotting. |
| Figure 2 | PBP `zvalley_1Darray_ctc<ctc>_<energy>MeV.txt` files plus matching `zpeak_1Darray` files for normalization. |
| Figure 5 | SOBP peak, valley, and PVDR text profiles for the panel cases listed in `data/data_dictionary.md`. The peak and valley profiles are expected to be final processed profiles scaled consistently with the manuscript normalization. |

No public CSV figure-source layer is generated or published. Public plotting code reads processed `.txt` files directly.

## 5. Excluded Files and Rationale

The public repository excludes:

- Raw and intermediate 3D dose arrays (`.npy`, `.mhd`, `.raw`, `.bin`, `.root`), because they can be large and are not necessary for readers who only need to run the public figure-generation scripts.
- Sidecar headers (`.header`) unless a future small example dataset is intentionally curated.
- Legacy README text files from the private data folder, because the public Markdown documentation replaces them with cleaned, publication-focused descriptions.
- Machine-specific or private path references from legacy scripts.
- `old_modules/`, slider scripts, workspace files, caches, and notebooks because they are exploratory or private workflow material rather than public reproducibility code.
- TOPAS run statistics and raw simulation outputs unless a small, non-private example is curated later.

## 6. Published Processed Text Files

The public repository publishes only processed text files that are needed by the selected public plotting scripts. Figure-specific data availability should be checked against `data/data_dictionary.md` before release.
