# Data Flow

This document summarizes the data flow reconstructed from the legacy analysis scripts. It intentionally uses generic path names such as `LEGACY_OUTPUT_ROOT` instead of copying local or machine-specific paths from the research folder.

## Terminology

- `ctc`: center-to-center distance between minibeams.
- `PVDR`: peak-to-valley dose ratio.
- `FWHM`: full width at half maximum.
- `PDD`: percentage depth dose.
- `PBP`: pristine Bragg peak.
- `SOBP`: spread-out Bragg peak.
- `bw`: beam width. In the legacy scripts the variable named `FWHM` is usually entered as tenths of a millimeter, for example `5` means `0.5 mm`.

## 1. Raw/Source Input Formats

The current proton pMBRT workflow is centered on 3D dose grids:

| Format | How it is used | Notes |
|---|---|---|
| `.npy` | Main input for the active proton scripts. Loaded with `numpy.load`. | The array is treated as a 3D dose grid. When `transpose=True`, legacy code applies `data.transpose(2, 1, 0)` so downstream arrays are ordered as `z, y, x`. |
| `.header` | Optional sidecar for `.npy` dose grids. | Read as CSV with columns `res_x`, `res_y`, and `res_z`; legacy code multiplies these values by 10 to obtain millimeters. If missing, default spacings are used. |
| `.mhd` | Supported by the base loader through SimpleITK. | The public repository does not include `.mhd` files because they are raw/intermediate dose-volume data. |
| TOPAS/statistics text | Used by `mergestatistics.py` to sum simulated primary counts from multiple statistics files. | This script prints a total; it does not generate a reusable summary table in the inspected workflow. |
| ROOT phase-space files | `utilities/readROOT.py` can read a `PhaseSpace` tree with `uproot`. | Not part of the prioritized figure-source data flow. |
| Older ion/PDD `.npy` profiles and RBE CSVs | `inspect_PDD.py` reads `zprofilepeak_*`, `zprofilevalley_*`, `LETpeak_*`, `LETvalley_*`, and RBE-vs-LET CSV files. | This appears to belong to an older ion/LET workflow, not the proton pMBRT figure-source data prioritized here. |

## 2. Intermediate Data Generated by Preprocessing

The active proton workflow uses derived arrays and text files before plotting:

| Step | Script/function | Input | Output |
|---|---|---|---|
| Create multi-beam dose grid | `create_multibeams.py` | `protonsinglebeam-Dose.npy` | `ctc{label}/protonmultibeam-Dose.npy` |
| Load profiles | `utilities/getprofile.py` and wrappers | `.npy` or `.mhd` dose grid | In-memory `zprofile_peak`, `zprofile_valley`, masked `zxprofile`, masked `zyprofile` |
| Save single-beam peak depth profile | `utilities/save_zprofile.py` | single-beam dose grid | `zpeak_singlebeam_{energy}MeV.txt` |
| Fit single-beam FWHM | `fit_singlebeam_FWHM.py` | single-beam lateral profiles | `{energy}MeV/FWHM_singlebeam_{energy}MeV.txt` |
| Fit multi-beam FWHM | `fit_FWHM.py` | multi-beam lateral profiles | `FWHM_1DGRID_ctc{ctc}_{energy}MeV.txt` or `FWHM_2DGRID_ctc{ctc}_{energy}MeV.txt` |
| Compute PVDR | `inspect_PVDRvsctc_toreview.py`, `Slider_PVDR.py` | in-memory peak and valley depth profiles | PVDR arrays computed as `z_peak / z_valley`; no persistent PVDR file was found |
| Compute 1D homogeneity | `inspect_1Dhomogeneity.py` | masked x profiles | `ctc{ctc}_homogeneity.npy`, a two-row array containing energy and pass/fail flags |
| Compute 1D homogeneity and relative standard deviation | `inspect_1Dhomog_and_std.py`, `inspect_xdeviation.py` | masked x profiles | homogeneity `.npy`; `inspect_xdeviation.py` writes `ctc{ctc}_std.npy` |
| Compute 2D lateral homogeneity | `inspect_2Dlateralhomogeneity.py` | masked x and y profiles | `ctc{ctc}_homogeneity_overxy.npy`, a two-row array containing energy and 2D pass/fail flags |

The multi-beam creator and profile loaders use center-to-center distance labels as tenths of a millimeter, for example `ctc12` for `1.2 mm`.

## 3. Processed Numerical Data Used for Plotting

| `mergestatistics.py` | Prints summed primary counts; no CSV or table output was found |

## 4. Files Needed to Populate the Public CSVs

To populate the processed CSVs in `data/figure_source_data/`, the following legacy processed outputs are needed:

| Public CSV | Required legacy outputs |
|---|---|
| `fig_fwhm_vs_energy.csv` | `FWHM_singlebeam_{energy}MeV.txt`, `FWHM_1DGRID_ctc{ctc}_{energy}MeV.txt`, `FWHM_2DGRID_ctc{ctc}_{energy}MeV.txt`, and metadata for setup, beam width, ctc, energy, and depth spacing |
| `fig_pvdr_vs_depth_or_ctc.csv` | Either saved peak/valley depth profiles or the dose grids needed to recreate `z_peak` and `z_valley`; inspected scripts do not persist PVDR arrays |
| `fig_homogeneity.csv` | `ctc*_homogeneity.npy`, `ctc*_homogeneity_overxy.npy`, and `ctc*_std.npy` |
| `summary_metrics.csv` | Curated values drawn from the FWHM, PVDR, homogeneity, and statistics outputs; the inspected scripts do not define or write a complete summary metrics table |

After the initial code inspection, a partial legacy dataset was provided outside the public repository. The subset is organized as `PBP_paperdataset/FWHM5` and includes `bw = 0.5 mm` PBP data. The public CSVs were populated from the available processed outputs in that subset:

- processed FWHM text files: `FWHM_singlebeam_*`, `FWHM_1Darray_ctc*`, and `FWHM_2Darray_ctc*`;
- processed PVDR text files: `PVDR_2Darray_*`;
- processed z-peak text files used for provenance inspection;
- homogeneity/min-max summary dictionaries: `dose_min_max_1Darray_dictionary.txt` and `dose_min_max_2Darray_finalversion.txt`;
- BEDR-related threshold dictionary: `dose_BEDR_1Darray_dictionary.txt`, which stores only a binary threshold flag, not a numeric BEDR value.

Raw/intermediate `.npy` dose grids in the partial subset were inspected for data-flow context but were not copied into the public repository. The small processed `.txt` files from this subset are copied under `data/processed_text/` because they are derived outputs, are small enough for GitHub, and can support future static visualization.

## 5. Excluded Files and Rationale

The public repository excludes:

- Raw and intermediate 3D dose arrays (`.npy`, `.mhd`, `.raw`, `.bin`, `.root`), including the partial raw/intermediate dose grids supplied with `PBP_paperdataset/FWHM5`, because they can be large and are not necessary for readers who only need to replot representative figure-source data.
- Sidecar headers (`.header`) unless a future small example dataset is intentionally curated.
- Legacy README text files from the private data folder, because the public Markdown documentation replaces them with cleaned, publication-focused descriptions.
- Machine-specific or private path references from legacy scripts.
- `old_modules/`, slider scripts, workspace files, caches, and notebooks because they are exploratory or private workflow material rather than public reproducibility code.
- TOPAS run statistics and raw simulation outputs unless a small, non-private example is curated later.

## 6. Published Processed CSV Files

The clean public repository defines these processed CSVs as the primary figure-source data:

- `data/figure_source_data/fig_fwhm_vs_energy.csv`
- `data/figure_source_data/fig_pvdr_vs_depth_or_ctc.csv`
- `data/figure_source_data/fig_homogeneity.csv`
- `data/figure_source_data/summary_metrics.csv`

These files contain long/tidy processed figure-source data extracted from the available partial `PBP_paperdataset/FWHM5` subset. They are representative, not complete coverage of all manuscript simulations.

The repository also includes `data/processed_text/PBP_paperdataset/FWHM5/`, a small processed-text provenance layer containing 403 `.txt` files from the same subset. These are lower-level processed outputs, not raw dose-volume arrays.

## Reproducibility Gaps Found During Inspection

- The public CSVs are based on a partial `bw = 0.5 mm` PBP subset, not the complete manuscript dataset.
- Some current legacy scripts depend on processed outputs that were not present in the supplied subset.
- Some scripts contain call/signature drift around `Get_profile_updated`, whose active constructor requires a `monobeam` argument.
- `inspect_1Dhomogeneity.py`, `inspect_1Dhomog_and_std.py`, `inspect_2Dlateralhomogeneity.py`, and `inspect_xdeviation.py` import `get_ctc_values_to_check_homogeneity`, but the active `utilities/functions.py` does not define it.
- The supplied partial subset contains processed PVDR text files labeled `PVDR_2Darray_*`; it does not include peak and valley dose profiles for every PVDR row, so the public PVDR CSV leaves `peak_dose_Gy` and `valley_dose_Gy` blank where only PVDR values are available.
- No numeric BEDR formula was found in the inspected scripts. The supplied `dose_BEDR_1Darray_dictionary.txt` stores `D_BP>0.6D_entrance` threshold flags only, so `summary_metrics.csv` includes `BEDR_threshold_flag` and leaves numeric `BEDR` blank.
- Some PVDR files in the supplied subset are stored under a parent energy folder that differs from the filename energy; extraction uses the filename energy and notes this condition where applicable.

## CSV Extraction Command

The processed CSV files can be rebuilt from the public processed-text layer or from a private legacy subset with:

```bash
python scripts/build_figure_source_data_from_legacy.py data/processed_text

# or, for maintainers with the private full/partial data folder:
python scripts/build_figure_source_data_from_legacy.py /path/to/pMBRTData
```

The script writes only processed CSVs and records relative source-file provenance; it does not copy raw arrays into the public repository.
